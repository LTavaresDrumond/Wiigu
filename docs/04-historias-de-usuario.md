# Historias de Usuario

## Finalidade

Este documento registra as historias de usuario do Wiigu. As historias descrevem os servicos esperados do sistema em linguagem centrada no usuario.

## HU-01: Cadastro de usuario

Como visitante, eu quero criar uma conta, para acessar os recursos do sistema.

## HU-02: Login

Como usuario cadastrado, eu quero entrar no sistema com minhas credenciais, para acessar meus projetos e quadros.

## HU-02A: Login com Google

Como usuario cadastrado, eu quero entrar no sistema com minha conta Google, para acessar meus projetos sem criar nova senha local.

## HU-03: Logout

Como usuario autenticado, eu quero sair do sistema, para encerrar minha sessao com seguranca.

## HU-04: Criar projeto

Como usuario autenticado, eu quero criar um projeto, para organizar quadros relacionados a um mesmo trabalho.

## HU-05: Gerenciar projetos

Como usuario autenticado, eu quero consultar, editar e excluir projetos, para manter minha area de trabalho atualizada.

## HU-06: Criar quadro Kanban

Como gestor do projeto, eu quero criar um quadro Kanban com as colunas A FAZER, FAZENDO e FEITO, para acompanhar visualmente as atividades do projeto.

## HU-07: Gerenciar quadro Kanban

Como gestor do projeto, eu quero consultar, editar e excluir quadros, para manter a organizacao do projeto coerente com o trabalho realizado.

## HU-08: Criar raia

Como gestor do projeto, eu quero criar raias no quadro, para separar atividades por frente de trabalho, modulo ou responsavel.

## HU-09: Gerenciar raias

Como gestor do projeto, eu quero consultar, editar e excluir raias, para reorganizar o quadro quando o projeto mudar.

## HU-10: Criar cartao

Como usuario autenticado, eu quero criar cartoes de atividade, para registrar tarefas com nome, responsavel, prazo, prioridade e descricao.

## HU-11: Gerenciar cartoes

Como usuario autenticado, eu quero consultar, editar e excluir cartoes, para manter as informacoes das atividades atualizadas.

## HU-12: Mover cartao entre colunas

Como responsavel por atividade, eu quero mover um cartao entre colunas, inclusive por arrastar e soltar, para representar a evolucao da tarefa no fluxo Kanban.

## HU-13: Mover cartao entre raias

Como gestor do projeto, eu quero mover um cartao entre raias, inclusive por arrastar e soltar, para reorganizar a atividade sem perder seu estado atual.

## HU-14: Definir limite WIP

Como gestor do projeto, eu quero definir limite WIP por coluna, para controlar a quantidade de trabalho em progresso.

## HU-15: Visualizar alerta de WIP

Como usuario do quadro, eu quero visualizar quando uma coluna atingir ou ultrapassar o limite WIP, para perceber gargalos no fluxo.

## HU-15A: Bloquear movimentacao acima do WIP

Como gestor do projeto, eu quero impedir novas entradas em uma coluna que ja atingiu o limite WIP, para manter a regra de fluxo combinada.

## HU-16: Visualizar metricas Kanban

Como gestor do projeto, eu quero visualizar lead time, cycle time, throughput e WIP, para avaliar o desempenho do fluxo de trabalho.
