# Wiigu

Projeto academico de Engenharia de Software para desenvolvimento do sistema Wiigu, uma ferramenta de apoio a gestao visual de projetos por Kanban.

O repositorio organiza tres frentes de trabalho:

- `docs/`: artefatos textuais exigidos na entrega.
- `prompts/`: roteiros internos de apoio para manter consistencia entre documentacao, UX e desenvolvimento.
- `prototype/`: codigo do prototipo funcional.
- `ux/`: storyboards, wireframes e especificacoes de interface.

## Linha de trabalho

O projeto usa um processo iterativo e incremental. A gestao do trabalho e feita em Kanban no Agiflow, com tarefas agrupadas por frente: gerenciamento, requisitos, arquitetura, UX, prototipo, testes e entrega.

As decisoes e documentos devem usar vocabulario alinhado a Engenharia de Software, especialmente requisitos, arquitetura, projeto, construcao, testes, qualidade, configuracao e gerenciamento de projeto.

## Ordem de execucao

1. Gerenciamento do projeto e guia de escrita.
2. Vision & Scope.
3. Requisitos funcionais, historias de usuario e requisitos nao funcionais.
4. Architecture Notebook e projeto fisico do banco.
5. Storyboards e wireframes.
6. Prototipo funcional.
7. Testes, infraestrutura, video e ZIP final.

## Observacao sobre Kanban

O projeto usa dois sentidos de Kanban:

- Kanban de gerenciamento do trabalho academico: quadro no Agiflow, usado como evidencia de planejamento e acompanhamento.
- Kanban implementado no prototipo: quadros do Wiigu com as colunas obrigatorias `A FAZER`, `FAZENDO` e `FEITO`.

Essa separacao deve aparecer nos documentos para evitar confusao entre o metodo usado pela equipe e a funcionalidade do sistema.
